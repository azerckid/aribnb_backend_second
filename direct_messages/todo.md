# Direct Messages Enhancements

## 목표
- 메시지에 **읽음 여부(read status)** 를 추가하여 사용자가 읽은 메시지를 구분할 수 있도록 한다.
- 메시지에 **첨부 미디어(사진/영상 등)** 를 연결하여 대화 중 파일을 주고받을 수 있도록 한다.

## 작업 순서
1. **모델 확장 기획**
   - `Message` 모델에 `is_read` Boolean 필드와 첨부용 FK/ManyToMany(예: `Photo`, `Video`) 구조를 정의한다.
   - 필요한 경우 새 미디어 유형(예: 일반 파일)을 고려하고, 스토리지 설정을 검토한다.

2. **데이터베이스 마이그레이션 준비**
   - 변경된 `Message` 모델에 맞춰 마이그레이션을 생성한다.
   - 기존 데이터에 기본값(미읽음/첨부 없음)을 설정할 수 있도록 계획한다.

3. **관리자 및 폼 업데이트**
   - Django Admin에서 읽음 여부와 첨부 미디어를 관리할 수 있도록 `MessageAdmin`을 수정한다.
   - 필요하다면 사용자-facing 폼/serializer에서도 입력을 받을 수 있도록 확장한다.

4. **비즈니스 로직/서비스 계층 적용**
   - 메시지 읽음 처리 로직(예: 메시지를 조회할 때 `is_read` 업데이트)을 구현한다.
   - 첨부 미디어 업로드 및 연결을 처리하는 유틸리티/서비스 코드를 작성한다.

5. **테스트 및 검증**
   - 유닛 테스트/통합 테스트를 통해 읽음 상태와 첨부 기능이 올바르게 동작하는지 확인한다.
   - 관리자/사용자 인터페이스에서 기능을 직접 확인하고 QA를 진행한다.

6. **문서화 및 배포 준비**
   - README 또는 개발 문서에 변경 사항을 기록하고 사용 방법을 정리한다.
   - 마이그레이션/환경 설정 등의 배포 단계 체크리스트를 업데이트한다.
